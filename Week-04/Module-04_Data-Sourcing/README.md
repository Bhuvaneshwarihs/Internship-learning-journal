# Week-04: Data Sourcing

## Module 4 — Data Collection and Web Scraping

### Overview
This module focuses on different techniques for collecting data from online sources. It covers web scraping, APIs, PDF extraction, and automated data collection workflows.

---

# Topics Covered

### Data Sourcing
- Data sourcing involves collecting data from websites, APIs, documents, and online databases for analysis or machine learning tasks.
- Reliable data collection methods ensure datasets are accurate, structured, and usable for further processing.

### Scraping with Excel
- Excel can be used to import web data using built-in tools such as Power Query.
- This allows users to extract tables or structured information directly from web pages.

### Scraping with Google Sheets
- Google Sheets provides functions like `IMPORTHTML` and `IMPORTXML` to extract data from websites.
- These functions allow automatic updates when the source webpage changes.

### BBC Weather API with Python
- APIs provide structured access to data through HTTP requests.
- Python libraries like `requests` can be used to retrieve weather data and process it programmatically.

### Scraping IMDb with JavaScript
- JavaScript can extract webpage data by interacting with the DOM elements.
- Tools like browser developer console or automation libraries help collect movie ratings and details.

### Nominatim API with Python
- The Nominatim API allows geocoding by converting location names into geographic coordinates.
- Python scripts can retrieve latitude and longitude for mapping and location analysis.

### Wikipedia Data with Python
- Wikipedia data can be accessed using APIs or scraping tools such as BeautifulSoup.
- Python can extract article text, tables, and structured information from Wikipedia pages.

### Scraping PDFs with Tabula
- Tabula is a tool used to extract tables from PDF documents.
- It converts PDF tables into structured formats such as CSV for analysis.

### Convert PDFs to Markdown
- PDF documents can be converted into Markdown format to make text easier to edit and process.
- This helps organize extracted content for documentation or further analysis.

### LLM Website Scraping
- Large Language Models can assist in extracting structured information from web pages.
- LLMs can interpret webpage content and identify relevant data automatically.

### LLM Video Screen-Scraping
- AI models can extract information from video frames or screen recordings.
- This technique helps capture data displayed in videos that cannot be directly downloaded.

### Scheduled Scraping with GitHub Actions
- GitHub Actions can automate scraping scripts to run on scheduled intervals.
- This allows regular data updates without manual execution.

### Scraping emarketer.com
- Web scraping techniques can collect market research data from websites like eMarketer.
- Extracted information can include statistics, trends, and market insights.

### Scraping: Live Sessions
- Live scraping involves collecting data in real-time during events or updates.
- This approach is useful for monitoring dynamic websites or streaming data sources.

---

# Key Learnings
- Data can be collected from websites, APIs, PDFs, and online databases.
- Web scraping allows extraction of structured information from webpages.
- APIs provide reliable and organized access to data sources.
- Automation tools like GitHub Actions enable scheduled data collection.

-----

# Conclusion
This module highlights the importance of data sourcing for data science and AI projects. By combining web scraping, APIs, and automation tools, developers can build reliable pipelines for collecting and maintaining datasets.
